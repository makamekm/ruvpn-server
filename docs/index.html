Launch your own RUPN server

A public, Docker-first, single-instance server for generating one signed RUPN connection link. Built for operators who want a small, supervised, auditable deployment without private orchestration services.

Deploy in one command View source
docker run --rm -it --name rupn-server -v rupn-server-state:/var/lib/rupn-server makame/rupn-server:latest

One container. One server.

RUPN Server intentionally avoids multi-tenant backend logic. The runtime defaults to the Telemost connection profile, validates environment variables, creates or loads a single room/key pair, starts one server process, and prints a JWT connection token.

1

Start Docker

Run the published image with a persistent volume so the generated connection link survives restarts.

docker run --rm -it --name rupn-server -v rupn-server-state:/var/lib/rupn-server makame/rupn-server:latest
2

Copy JWT

The container prints RUPN_CONNECT_JWT. Use this value in the client app. Do not share the raw URI publicly.

RUPN_CONNECT_JWT=eyJhbGciOiJIUzI1NiIs...
3

Keep it private

The Docker volume stores the server key. Treat it as private infrastructure state.
